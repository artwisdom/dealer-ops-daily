# Dealer Ops Daily

Automated daily beehiiv newsletter for U.S. auto dealership operators. AI-edited, human-reviewed.

> The 5-minute morning briefing for the people running U.S. car dealerships — F&I directors, desk managers, used-car managers, compliance leads, and floor GMs.

**Status:** Phases 0–5 complete (pipeline + growth automation + 6 self-improvement loops + Sunday Opus meta-issue + launch tooling, 60 passing tests). Pre-launch — needs API keys.

---

## What this repo does

Every weekday at 05:30 UTC, GitHub Actions runs `pipeline/run.py`:

1. **Ingest** — pulls candidate stories from 70+ RSS sources defined in `sources.yaml`
2. **Rank** — Claude scores by importance/novelty and assigns a section
3. **Draft** — Claude writes a publication-ready issue using `prompts/system-prompt-v1.md`
4. **Image** — Flux Schnell generates a hero image (Pexels fallback)
5. **Affiliates** — picks the best-fit affiliate from `data/affiliates.json` (or Supabase)
6. **Publish** — POSTs to beehiiv API and schedules send for 06:00 ET

Then nightly at 23:13 UTC, a second workflow pulls open/click stats back into `data/analytics.json` so the next morning's prompt sees yesterday's performance.

## Quick start

```bash
# Install
pip install -r requirements.txt

# Generate offline test data
python -m pipeline.ingest --seed-fixture

# Run the full pipeline in dry mode (no API calls, no sends)
python -m pipeline.run --dry-run

# Run the test suite
python -m pytest tests/
```

Going live requires API keys in your `.env` (or as GitHub Actions secrets) — see [SETUP.md](SETUP.md).

## Editorial guardrails (hard rules)

Enforced in `pipeline/draft.py` via a self-check the model must pass before output is accepted:

- ≥2 sources per non-trivial claim
- No financial advice / stock picks
- No partisan political commentary
- No medical claims
- Quotes ≤25 words per source
- All numbers cited

Violations → pipeline refuses to publish and opens a GitHub Issue.

Full editorial standards: [research/06-editorial-standards.md](research/06-editorial-standards.md).

## Architecture decisions

- **Runtime:** GitHub Actions, not Claude Routines (see [research/07-runtime-architecture.md](research/07-runtime-architecture.md))
- **Models:** Sonnet 4.6 for daily issues, Opus 4.6 for the Sunday meta-issue
- **State:** local JSON files committed to repo (analytics, audit trail). Supabase + Sheets are optional upgrades.
- **No-key path:** every module has a dry-run / fixture mode so tests run without API keys
